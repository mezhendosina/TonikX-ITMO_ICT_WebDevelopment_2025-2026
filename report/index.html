5 Самых Популярных Алгоритмов Сортировки

1. Пузырьковая Сортировка (Bubble Sort)

Пузырьковая сортировка - это простой алгоритм сортировки, который многократно проходит по списку, сравнивает соседние элементы и меняет их местами, если они находятся в неправильном порядке. Процесс повторяется до тех пор, пока список не будет отсортирован.

Особенности:

  • Временная сложность: O(n²) во всех случаях
  • Пространственная сложность: O(1)
  • Стабильность: Стабильный алгоритм
  • Адаптивность: Может быть адаптивным (оптимизированным для частично отсортированных данных)

Когда использовать:

  • Образовательные цели
  • Очень маленькие наборы данных
  • Когда простота реализации важнее эффективности

Реализация на Python:

def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        # Флаг для оптимизации - если не было обменов, массив отсортирован
        swapped = False
        for j in range(0, n - i - 1):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                swapped = True
        # Если не было обменов, выходим
        if not swapped:
            break
    return arr

2. Сортировка Выбором (Selection Sort)

Сортировка выбором делит входной список на две части: отсортированную подсписку элементов, которая строится слева направо, и подсписок оставшихся несортированных элементов, который занимает остальную часть массива. Изначально отсортированная подсписок пуста, а несортированная подсписок - весь входной массив. Алгоритм находит наименьший (или наибольший) элемент в несортированной подсписке и меняет его местами с крайним левым элементом несортированной подсписки, тем самым расширяя отсортированную подсписок на один элемент вправо.

Особенности:

  • Временная сложность: O(n²) во всех случаях
  • Пространственная сложность: O(1)
  • Стабильность: Нестабильный алгоритм
  • Адаптивность: Не адаптивный

Когда использовать:

  • Когда память ограничена
  • Когда стоимость записи значительно выше стоимости чтения
  • Для небольших наборов данных

Реализация на Python:

def selection_sort(arr):
    n = len(arr)
    for i in range(n):
        # Находим индекс минимального элемента в оставшейся части массива
        min_idx = i
        for j in range(i + 1, n):
            if arr[j] < arr[min_idx]:
                min_idx = j
        # Меняем местами текущий элемент с минимальным
        arr[i], arr[min_idx] = arr[min_idx], arr[i]
    return arr

3. Сортировка Вставками (Insertion Sort)

Сортировка вставками строит окончательный отсортированный массив (или список) по одному элементу за раз. Для каждого элемента алгоритм находит правильную позицию в уже отсортированной части массива и вставляет туда элемент.

Особенности:

  • Временная сложность:
  • Лучший случай: O(n) (если массив уже отсортирован)
  • Средний и худший случаи: O(n²)
  • Пространственная сложность: O(1)
  • Стабильность: Стабильный алгоритм
  • Адаптивность: Адаптивный алгоритм

Когда использовать:

  • Небольшие наборы данных
  • Частично отсортированные данные
  • Онлайн-сортировка (когда данные поступают по одному)
  • Как финальная стадия гибридных алгоритмов (например, Quicksort)

Реализация на Python:

def insertion_sort(arr):
    for i in range(1, len(arr)):
        key = arr[i]
        j = i - 1
        # Перемещаем элементы arr[0..i-1], которые больше key, на одну позицию вперед
        while j >= 0 and arr[j] > key:
            arr[j + 1] = arr[j]
            j -= 1
        arr[j + 1] = key
    return arr

4. Сортировка Слиянием (Merge Sort)

Сортировка слиянием - это алгоритм "разделяй и властвуй". Он делит входной массив на две половины, рекурсивно сортирует каждую половину и затем объединяет две отсортированные половины.

Особенности:

  • Временная сложность: O(n log n) во всех случаях
  • Пространственная сложность: O(n)
  • Стабильность: Стабильный алгоритм
  • Адаптивность: Не адаптивный

Когда использовать:

  • Когда требуется стабильная сортировка
  • Когда нужна гарантированная производительность O(n log n)
  • Для больших наборов данных
  • Когда данные не помещаются в память (внешняя сортировка)

Реализация на Python:

def merge_sort(arr):
    if len(arr) <= 1:
        return arr

    # Делим массив на две половины
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])

    # Объединяем отсортированные половины
    return merge(left, right)

def merge(left, right):
    result = []
    i = j = 0

    # Сравниваем элементы из обеих половин и объединяем их в отсортированном порядке
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1

    # Добавляем оставшиеся элементы
    result.extend(left[i:])
    result.extend(right[j:])

    return result

5. Быстрая Сортировка (Quick Sort)

Быстрая сортировка - это еще один алгоритм "разделяй и властвуй". Он выбирает "опорный" элемент и разделяет массив на два подмассива: элементы меньше опорного и элементы больше опорного. Затем рекурсивно сортирует подмассивы.

Особенности:

  • Временная сложность:
  • Лучший и средний случаи: O(n log n)
  • Худший случай: O(n²) (когда опорный элемент всегда наименьший или наибольший)
  • Пространственная сложность: O(log n) (из-за рекурсии)
  • Стабильность: Нестабильный алгоритм
  • Адаптивность: Не адаптивный

Когда использовать:

  • В общем случае для больших наборов данных
  • Когда дополнительная память ограничена
  • В стандартных библиотеках многих языков программирования

Реализация на Python:

def quick_sort(arr):
    if len(arr) <= 1:
        return arr

    pivot = arr[len(arr) // 2]  # Выбираем средний элемент как опорный
    left = [x for x in arr if x < pivot]    # Элементы меньше опорного
    middle = [x for x in arr if x == pivot] # Элементы равные опорному
    right = [x for x in arr if x > pivot]   # Элементы больше опорного

    return quick_sort(left) + middle + quick_sort(right)

# Альтернативная реализация in-place (на месте)
def quick_sort_inplace(arr, low=0, high=None):
    if high is None:
        high = len(arr) - 1

    if low < high:
        # Разделяем массив и получаем индекс опорного элемента
        pivot_index = partition(arr, low, high)

        # Рекурсивно сортируем элементы до и после опорного
        quick_sort_inplace(arr, low, pivot_index - 1)
        quick_sort_inplace(arr, pivot_index + 1, high)

    return arr

def partition(arr, low, high):
    pivot = arr[high]  # Выбираем последний элемент как опорный
    i = low - 1  # Индекс меньшего элемента

    for j in range(low, high):
        if arr[j] <= pivot:
            i += 1
            arr[i], arr[j] = arr[j], arr[i]

    arr[i + 1], arr[high] = arr[high], arr[i + 1]
    return i + 1

Заключение

Выбор алгоритма сортировки зависит от конкретных требований задачи: - Для небольших наборов данных или частично отсортированных данных хорошо подходит сортировка вставками. - Для стабильной сортировки с гарантированной производительностью O(n log n) выбирают сортировку слиянием. - Для общих случаев с большими наборами данных часто используется быстрая сортировка. - Пузырьковая сортировка и сортировка выбором в основном используются в образовательных целях.
